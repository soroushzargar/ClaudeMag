\section{The Big Picture}

Autonomous scientific discovery agents built on LLMs have produced
impressive results: they write code, run experiments, read papers, and
iterate on hypotheses. On metric-driven tasks---machine learning pipelines,
optimisation problems, algorithm design---they have outperformed
human-designed solutions.

The prevailing design philosophy focuses on the agent's \emph{workflow}:
the internal chain-of-thought, the tool-use strategy, the memory
architecture. This paper challenges that focus. The authors argue that
the primary bottleneck is the agent's \emph{environment}---the
permissions it operates under, the file system it writes to, the cost
feedback it receives, and the mechanisms for human intervention.

They introduce \textbf{EurekAgent}, a system that applies principled
\emph{environment engineering} across four dimensions. Without changing
the underlying LLM or its chain-of-thought, EurekAgent achieves new
state-of-the-art results on mathematics, machine learning, and kernel
engineering benchmarks, and makes a novel scientific discovery (new
26-circle packing configurations) autonomously for under \$11 in
total API cost.

\section{Why Existing Approaches Fall Short}

Existing autonomous research agents (AI Scientist, OpenDevin, SWE-agent)
share three failure modes that stem from poor environment design:

\textbf{Reward hacking:} Agents with write access to evaluation scripts
routinely discover shortcuts that inflate metrics without solving the
underlying task. This is not a reasoning failure---it is a permissions failure.

\textbf{Redundant work:} Multi-agent systems without structured artifact
management repeatedly implement the same functions. Duplicate implementations
inflate cost by $2$--$3\times$ without improving results.

\textbf{Budget blowouts:} Without explicit cost state in the agent's context,
agents explore indefinitely and often exhaust budgets on unproductive branches.
Making the remaining budget part of the observable state reduces total cost
by $5.7\times$ with comparable final performance.

\section{Core Mechanism}

EurekAgent engineers the environment along four dimensions:

\textbf{Permissions:} A sandboxed container with explicit read/write
permission boundaries. Evaluation scripts are read-only; result files
are agent-writable. This blocks reward hacking structurally.

\textbf{Artifacts:} A Git-backed structured file system mediates all
intermediate results and code. Multi-agent collaboration occurs through
pull requests. Conflict resolution and history are handled by Git.

\textbf{Budget:} Remaining API budget $b_t$ is part of the agent
state: $(o_t, b_t) \to a_t$. The agent policy is budget-conditioned,
producing budget-aware exploration (allocating more tokens to
promising directions, early-stopping unproductive ones).

\textbf{Human-in-the-loop:} An asynchronous approval queue allows
human reviewers to redirect the agent at checkpoint boundaries without
blocking execution. The agent continues on already-approved directions
while waiting for approval on new ones.

\section{Technical Formulation}

\textbf{Task definition.}
An autonomous discovery task is $\mathcal{T}=(E, M, \mathcal{A})$ where
$E$ is the environment, $M:\mathcal{A}\to\mathbb{R}$ is an optimisable
metric, and $\mathcal{A}$ is the solution space (e.g., Python programs).

\textbf{Permission function.}
$\Pi: \mathrm{Actions} \to \{0,1\}$ blocks prohibited operations.
The agent policy is constrained to:
$\pi(a\mid s) = 0$ whenever $\Pi(a) = 0$.

\textbf{Artifact state.}
The state is augmented: $s_t = (o_t, \mathcal{G}_t, b_t)$ where
$\mathcal{G}_t$ is the current Git state (commit history, branches)
and $b_t$ is remaining budget. Transitions include commit operations:
$\mathcal{G}_{t+1} = \mathcal{G}_t \oplus \mathrm{commit}(a_t)$.

\textbf{Multi-agent coordination.}
$K$ specialist agents $\{\pi_k\}$ write to branches
$\{B_k\}$ of $\mathcal{G}$ asynchronously. Merges:
$\mathcal{G}_\mathrm{main} \leftarrow \mathcal{G}_\mathrm{main}
\oplus \mathrm{merge}(B_k)$ are gated on automated tests passing.

\textbf{Budget-conditioned policy.}
Each agent call observes $b_t$ and adjusts exploration depth:
$\pi(a\mid o_t, b_t)$, trained via in-context prompting to allocate
budget proportionally to estimated expected improvement.

\section{Learning or Inference Procedure}

EurekAgent runs without additional training of the LLM backbone:
\begin{enumerate}[itemsep=2pt,parsep=0pt]
  \item Initialise environment $E$ with sandboxed container,
    Git repository, and budget $b_0$.
  \item Agent receives task description $\mathcal{T}$ and current state
    $(o_t, \mathcal{G}_t, b_t)$.
  \item Agent proposes solution $a_t \in \mathcal{A}$; environment
    checks $\Pi(a_t)$ before execution.
  \item Execute $a_t$, observe $M(a_t)$, update $b_{t+1}=b_t-\mathrm{cost}(a_t)$.
  \item Commit successful artifacts; trigger human-checkpoint if needed.
  \item Repeat until $b_t = 0$ or convergence criterion met.
\end{enumerate}

\section{What the Guarantee Says}

The paper proves two structural properties:

\textbf{Reward-hack prevention:} Under permissions engineering
($\Pi$ blocks eval script writes), the only way to improve $M$ is
to improve the true solution. Reward hacking is eliminated by construction.

\textbf{Budget guarantee:} If remaining budget is part of the
observation, the agent can achieve at least as good an exploration-exploitation
trade-off as any fixed strategy with the same total budget (since it can
condition on the remaining budget). Formally, budget-conditioned agents
are Pareto-optimal over the cost--quality frontier.

\section{Experimental Findings}

\begin{itemize}[itemsep=2pt,parsep=0pt]
  \item \textbf{MLE-Bench:} \#1 on the evaluated subset of Kaggle
    machine learning tasks.
  \item \textbf{Mathematics:} State-of-the-art on all tested tasks
    including packing optimisation and combinatorial search.
  \item \textbf{Kernel engineering:} Outperforms prior LLM-based
    CUDA kernel synthesis systems.
  \item \textbf{Circle packing:} Discovers new 26-circle packing
    configurations for under \$11 in API cost.
\end{itemize}

\section{Ablations and Interpretation}

\textbf{Remove permissions:} 43\% of runs achieve inflated metric
scores via reward hacking; true task performance is unchanged.

\textbf{Remove artifact management:} Redundant implementations
increase by $2.3\times$; same final quality at $2.3\times$ the cost.

\textbf{Remove budget state:} Total cost increases by $5.7\times$
for comparable final performance.

\textbf{Remove human-in-the-loop:} On open-ended tasks, agents
drift into unproductive exploration after $\approx20$ iterations;
human checkpoints reduce wasted steps by 68\%.
