# BiSCo-LLM: Lookup-Free Binary Spherical Coding for Extreme Low-Bit Large Language Model Compression

**Authors:** Yuantian Shao, Peisong Wang, Zhilei Liu, Chuangyi Li, Yuanteng Chen, Pengcheng Xie, Yiwu Yao, Zhihui Wei, Jian Cheng
**Affiliation:** Institute of Automation, Chinese Academy of Sciences (CASIA); University of Chinese Academy of Sciences; Huawei Technologies
**Venue:** arXiv preprint
**arXiv:** 2607.08643
**URL:** https://arxiv.org/abs/2607.08643

---

## Headline Finding

Deploying large language models on memory-constrained hardware is bottlenecked by weight storage: even 4-bit quantization still demands tens of gigabytes for frontier models. BiSCo-LLM eliminates the lookup table overhead that plagues existing binary quantization schemes by mapping weight vectors onto a high-dimensional sphere and using binary spherical codes as weight representations — achieving competitive language modeling quality at sub-1-bit effective precision without any decoding table at runtime.

---

## Key Findings (Inverted Pyramid)

### Level 1 — Main Result
BiSCo-LLM introduces **Binary Spherical Coding (BSCo)**, a lookup-free quantization scheme that:
- Represents weight matrices using binary codewords on the unit hypersphere
- Eliminates lookup-table overhead entirely (zero additional memory for the codebook)
- Achieves competitive perplexity on WikiText-2 against INT2/INT4 quantization baselines at the same or lower storage budget
- Reduces LLaMA-3-8B memory footprint by up to 12× versus FP16 with less than 3 points of perplexity degradation

### Level 2 — Core Mechanism
Standard codebook-based quantization (e.g., AQLM, QuIP) maps weight blocks to entries in a learnable codebook. The codebook itself must be stored in memory, and decoding requires a lookup operation at each matrix-vector multiply — adding latency and memory overhead proportional to codebook size. Binary spherical coding sidesteps both costs: each weight column is encoded as a sum of binary ($\pm1$) basis vectors drawn from a pre-defined spherical frame, requiring only a fast sign-dot-product operation to decode. The spherical constraint ensures reconstruction error is geometrically minimal and uniformly distributed across the weight norm spectrum.

### Level 3 — Methodology
The compression pipeline proceeds as follows:
1. **Frame construction:** Compute a binary tight frame $\{b_k\}_{k=1}^K \subset \{+1,-1\}^d$ satisfying $\sum_k b_k b_k^\top = cI$ for a constant $c$
2. **Coding:** For each weight vector $w \in \mathbb{R}^d$, find binary coefficients $z \in \{0,1\}^K$ minimizing $\|w - \frac{1}{\sqrt{K}}\sum_k z_k b_k\|^2$
3. **Reconstruction:** At inference, $\hat{w} = \frac{1}{\sqrt{K}}\sum_k z_k b_k$ — computed by a simple inner product with binary vectors, implementable as XNOR-popcount operations

Fine-tuning with straight-through estimators (STE) allows end-to-end training-time optimization of $z$ given the fixed frame.

### Level 4 — Technical Details
The spherical tight frame construction solves:
$$\min_{\{b_k\} \subset \{+1,-1\}^d} \left\|\sum_{k=1}^K b_k b_k^\top - \frac{K}{d} I_d\right\|_F^2$$

This is a binary Grassmannian packing problem. The authors use a greedy frame-building algorithm that sequentially adds basis vectors maximizing frame orthogonality.

The STE gradient for binary coefficients $z_k \in \{0,1\}$:
$$\frac{\partial \mathcal{L}}{\partial z_k} \approx \frac{\partial \mathcal{L}}{\partial \hat{w}} \cdot \frac{b_k}{\sqrt{K}} \cdot \mathbf{1}[|z_k - 0.5| < 0.5]$$

The effective bit-rate of BiSCo-LLM is $\log_2(K)/d$ bits per weight, adjustable by varying the number of frame vectors $K$. At $K = d$ this gives approximately 1 bit per parameter; at $K = d/2$ approximately 0.5 bits per parameter.

---

## Why Existing Approaches Fall Short

Existing extreme quantization methods fall into two camps. (1) Codebook-based methods (AQLM, QuIP, VQ-LLM) achieve excellent reconstruction quality but require storing and accessing a codebook of up to 65,536 entries, creating an additional memory footprint of several hundred megabytes and lookup latency at every inference step. (2) Pure binary methods (BitNet, OneBit) represent weights as $\pm1$ scalars, making them lookup-free, but suffer from severe accuracy degradation because $\pm1$ cannot represent heterogeneous weight magnitudes. BiSCo-LLM achieves the quality of codebook methods and the zero-overhead runtime of binary methods by separating the representation structure (binary on a sphere) from the scalar magnitude (encoded in the frame normalization).

## Experimental Findings

| Method | Bits/weight | LLaMA-3-8B PPL (WikiText-2) | Memory (GB) |
|--------|-------------|----------------------------|-------------|
| FP16 (baseline) | 16 | 6.14 | 16.0 |
| GPTQ-4bit | 4 | 6.58 | 4.0 |
| AQLM-2bit | 2 | 7.21 | 2.0 |
| BitNet-1bit | 1 | 9.87 | 1.0 |
| **BiSCo-LLM (1bit)** | **1** | **7.73** | **1.0** |
| **BiSCo-LLM (0.5bit)** | **0.5** | **9.14** | **0.5** |

BiSCo-LLM at 1-bit precision matches AQLM at 2-bit precision within 0.5 perplexity points while using half the memory, and outperforms BitNet by 2.1 perplexity points at the same storage budget.

---

## Significance

The practical gap between the largest frontier LLMs and on-device deployment is primarily a memory bandwidth problem. BiSCo-LLM's lookup-free design eliminates the memory cost of codebooks and the latency cost of table lookups, making it the first extreme-quantization method that is simultaneously practical at runtime (no lookup overhead) and competitive in quality with codebook-based approaches. If binary spherical coding proves robust across model families and tasks, it could define a new efficiency frontier for mobile and edge deployment of language models.
